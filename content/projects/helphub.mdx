## 課題

問い合わせや依頼がチャット・口頭・メールに分散し、対応状況と履歴を追いにくい状態が続いていた。
再確認や引き継ぎのたびに過去経緯を探す負荷が発生し、対応品質のばらつきも課題になっていた。

## アプローチ

問い合わせをチケットとして一元管理し、状態遷移・履歴・添付情報を同じ画面で確認できる構成にした。
権限に応じた閲覧範囲を明確にし、運用ルールを画面設計とあわせて整理した。
実装は小さな単位でPR化し、CIで確認しながら段階的に改善を重ねた。

## 技術

- Next.js / TypeScript
- Supabase
- Vercel
- GitHub Actions

## 結果

対応状況と履歴の見通しが改善され、問い合わせ管理の運用判断をそろえやすくなった。
添付情報を含めた記録を残せるため、引き継ぎ時の確認コストも抑えやすくなった。
デモやソースの詳細は応募時にご案内します。

## 学び

管理項目だけでなく運用時の判断基準まで先に揃えると、後工程の手戻りを減らしやすい。
小さく改善して検証する進め方は、業務影響を抑えながら品質を上げるのに有効だった。
